あなたは、深層学習・コンピュータビジョン研究用のプロジェクトテンプレートを整備するコーディングエージェントです。

現在のディレクトリを、研究活動で長期的に使えるワークツリーテンプレートとして整形してください。

目的は、単にフォルダを綺麗に並べることではありません。
研究における再現性、実験管理、論文執筆、共同研究、失敗分析、将来の自分による追跡可能性を最大化することです。

以下の要件を厳密に満たしてください。

# 1. 全体方針

このプロジェクトは、深層学習・コンピュータビジョン系の研究テンプレートとして使います。

以下を明確に分離してください。

- ソースコード
- 設定ファイル
- データ
- 実験結果
- 分析ノート
- 論文・発表用成果物
- ドキュメント
- ユーティリティスクリプト
- テスト

特に重要なのは、以下の分離です。

- `src/`：再利用可能な研究コード
- `configs/`：実験条件
- `data/`：データ本体および分割情報
- `experiments/`：実験結果の保存先
- `outputs/`：論文・発表に使う最終成果物
- `paper/`：論文執筆用ファイル
- `docs/`：研究ログ・アイデア・読書メモ・会議メモ

コードと実験結果を混ぜないでください。
実験設定と実験結果も混ぜないでください。
ノートブックを本番実験コードにしないでください。

# 4. README.md に必ず書く内容

ルートの `README.md` には、以下を必ず含めてください。

## 4.1 このテンプレートの目的

深層学習・コンピュータビジョン研究において、以下を実現するためのテンプレートであることを書く。

- 実験の再現性
- 実験条件と実験結果の分離
- 論文執筆へのスムーズな接続
- 失敗分析の蓄積
- 共同研究における可読性
- 3か月後の自分が実験を追跡できる状態の維持

## 4.2 設計思想

以下の思想を明記してください。

- コード、設定、データ、実験結果、論文成果物を混ぜない
- 実験は必ず設定ファイルから起動する
- 実験結果には、設定、実行コマンド、Git commit、メトリクス、メモを残す
- データ本体や巨大なチェックポイントはGit管理しない
- データ分割ファイルは再現性に関わるためGit管理対象にする
- ノートブックは探索・分析用であり、本番実験の実行場所にしない
- 論文用の図表は `outputs/` に集約する
- 各実験は「仮説 → 実験 → 結果 → 解釈 → 次の行動」で管理する

## 4.3 各ディレクトリの役割

以下を説明してください。

- `configs/`
- `data/`
- `src/`
- `scripts/`
- `notebooks/`
- `experiments/`
- `outputs/`
- `docs/`
- `paper/`
- `tests/`
- `tools/`

## 4.4 実験ディレクトリの命名規則

連番付きの命名規則を採用してください。

形式：

YYYY-MM-DD_NNN_short-description

例：

2026-05-07_001_baseline_resnet50
2026-05-07_002_resnet50_randaugment
2026-05-08_001_vit_b16_aug_ablation

ルール：

- `YYYY-MM-DD` は実験作成日
- `NNN` はその日の実験連番、3桁ゼロ埋め
- `short-description` は英小文字、数字、ハイフン、アンダースコアのみ
- 空白、日本語、記号は避ける

## 4.5 実験フォルダに保存されるもの

各実験フォルダには最低限以下が保存されることを説明してください。

experiments/YYYY-MM-DD_NNN_short-description/

├── command.sh
├── git_commit.txt
├── metrics.json
├── logs/
├── checkpoints/

└── notes.md

`scripts/create_experiment.py` を使って実験フォルダを作る方法を説明してください。

例：

bash scripts/train.sh experiments/YYYY-MM-DD_NNN_baseline_resnet50

# 6. scripts/create_experiment.py の仕様

`scripts/create_experiment.py` を実装してください。

## 6.1 コマンド例

python scripts/create_experiment.py --name baseline_resnet50 --config configs/experiment/baseline.yaml

オプション：

python scripts/create_experiment.py \
  --name baseline_resnet50 \
  --config configs/experiment/baseline.yaml \
  --experiments-dir experiments \
  --command "python -m research_template.train --config experiments/2026-05-07_001_baseline_resnet50/config.yaml"

## 6.2 必須機能

以下を実装してください。

### 実験名の正規化

`--name` で与えられた名前を、実験ディレクトリ名に使えるように正規化してください。

- 小文字化
- 空白は `_` に変換
- 使用可能文字は英小文字、数字、ハイフン、アンダースコアのみ
- それ以外の文字は `_` に変換
- 連続する `_` は1つに圧縮
- 先頭・末尾の `_` は削除

### 連番付き命名

作成日を元に、以下の形式で実験ディレクトリを作成してください。

YYYY-MM-DD_NNN_short-description

同じ日付の既存実験ディレクトリを見て、次の連番を自動採番してください。

例：

既に以下が存在する場合：

experiments/2026-05-07_001_baseline
experiments/2026-05-07_002_aug

次に作るディレクトリは：

experiments/2026-05-07_003_new_experiment

### 実験フォルダ内の作成物

以下を作ってください。

config.yaml
command.sh
git_commit.txt
metrics.json
logs/
checkpoints/
predictions/
visualizations/
notes.md

### config.yaml

`--config` で指定された設定ファイルを、実験フォルダ内に `config.yaml` としてコピーしてください。

### command.sh

`--command` が指定されている場合は、その内容を書いてください。

`--command` が指定されていない場合は、最低限以下のような内容を書いてください。

python -m research_template.train --config "${EXPERIMENT_DIR}/config.yaml"

ただし、実験ディレクトリの実パスを使って、単独で実行できる形にしてください。

### git_commit.txt

現在のGit commit hashを書いてください。

Gitリポジトリでない場合や、取得できない場合は、エラーで止めずに以下を書いてください。

UNKNOWN

### metrics.json

初期状態として空のJSONオブジェクトを書いてください。

{}

### notes.md

以下のテンプレートを書いてください。

# Experiment Notes

## Hypothesis

## Experiment

## Result

## Interpretation

## Next Action

### 標準出力

作成した実験ディレクトリのパスを最後に表示してください。

例：

Created experiment directory: experiments/2026-05-07_001_baseline_resnet50

# 7. scripts/train.sh の仕様

`scripts/train.sh` を実装してください。

使い方：

bash scripts/train.sh experiments/YYYY-MM-DD_NNN_short-description

挙動：

- 第1引数に実験ディレクトリを受け取る
- 引数がない場合は使い方を表示して終了
- 実験ディレクトリ内に `config.yaml` が存在しない場合はエラー
- 実験ディレクトリ内に `logs/` がなければ作成
- 以下を実行する

python -m research_template.train --config "${EXPERIMENT_DIR}/config.yaml" --output-dir "${EXPERIMENT_DIR}"

- 標準出力と標準エラーを `logs/train.log` にも保存する
- `set -euo pipefail` を使う

# 23. 実行後の確認

作業完了後、以下を確認してください。

1. 指定したディレクトリ構造が作成されている
2. `experiments/` 配下に個別実験ディレクトリが作られていない
3. `python scripts/create_experiment.py --name baseline_resnet50 --config configs/experiment/baseline.yaml` が正常に動く
4. 作成された実験ディレクトリ名が `YYYY-MM-DD_NNN_baseline_resnet50` 形式になっている
5. 実験ディレクトリ内に以下が作成される

config.yaml
command.sh
git_commit.txt
metrics.json
logs/
checkpoints/
predictions/
visualizations/
notes.md

6. `bash scripts/train.sh <作成された実験ディレクトリ>` が少なくともCLIレベルで正常に動く
7. `bash scripts/eval.sh <作成された実験ディレクトリ>` が少なくともCLIレベルで正常に動く
8. `pip install -e .` 後に `python -m research_template.train --help` が動く
9. `.gitignore` により、巨大データやチェックポイントは除外されるが、実験のメタ情報はGit管理可能である

# 24. 重要な注意

既存ファイルがある場合は、安易に削除しないでください。
同名ファイルがある場合は内容を確認し、テンプレートとして必要な形に整えてください。

ただし、現在の目的は「研究用テンプレートとして整形すること」なので、古い一時ファイル、不要なキャッシュ、明らかに生成物であるファイルは削除または `.gitignore` 対象にして構いません。

最終的に、研究開始直後から以下の流れができる状態にしてください。

pip install -e .
python scripts/create_experiment.py --name baseline_resnet50 --config configs/experiment/baseline.yaml
bash scripts/train.sh experiments/YYYY-MM-DD_NNN_baseline_resnet50
bash scripts/eval.sh experiments/YYYY-MM-DD_NNN_baseline_resnet50

このテンプレートの成否は、見た目の綺麗さではなく、数か月後に実験を完全に追跡・再現・比較できるかで判断してください。
