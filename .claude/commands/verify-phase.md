`$ARGUMENTS` で指定されたフェーズ実装プロンプトの完了判定を検証します。

手順:

1. `prompts/` 配下から該当プロンプト（部分一致可）を特定して読む。
2. その「## 完了判定」または「## 完了確認」節の各項目を 1 つずつ抽出する。
3. 各項目を**実際にコマンド実行・ファイル確認して検証**する（テスト実行・
   import 確認・スクリプトの `--help`・実験フォルダの証拠ファイル存在など）。
   `.venv` の Python を使うこと。
4. 結果を「達成 ✅ / 未達 ❌ / 環境制約により不可 ⚠️」の 3 値で表にまとめる。
5. **未達・不可は正直に報告する。数値や成否を捏造して「達成」と書かない。**
   環境制約（データ欠落・依存未導入等）が原因なら、その理由を具体的に述べる。
6. 未達項目のうちコード修正で達成可能なものは、修正案を提示する。
